文件修改忘记改了什么怎么办?使用git

2019年05月30日 星期二, 发表于 北京

”—-你本来不该知道这些的，可是现在我们没有更好的工具，所以得这样弄一下……”

这里主要就是解决1个问题: 在平时凡是用到文本的时候，任何人都易犯错，那么如何看出一个文件我们改了什么?什么时候改的？

最快的方法是用man命令，每个命令，都有人写好了标准的使用手册就用man命令，还可以上网查阅相关资料，如高能所计算环境使用手册，现在STFW是十分方便的，也可以看下面的写的细了。

$ man git

这里介绍git的基本用法，另外还有github,gitpage,gitbook和其对应的国产软件将在之后再写写看。这是程序员们使用的基本工具，Linus发起的版本控制软件，只学学最简单的直接用。

git可以记录并显示文件修改历史，还可以进行不同版本的比较，如果需要可以退回上次的修改。这里所讲都是linux下的内容(简单!)，win的请自行百度。

(自己之前都没有写下来该怎么用，还经常忘记，今天又发生了代码误删的情况所以这回要把git学会了)

检查git: 首先是检查git的版本:

$git --version 
git version 1.8.2.1

好的，下一步是了解基本概念，什么是git(也可以跳过直接看使用): 在Git中，每个版本库都叫做一个仓库（repository），每个仓库可以简单理解成一个目录，这个目录里面的所有文件都通过Git来实现版本管理，Git都能跟踪并记录在该目录下所有内容(子目录递归包含)发生的所有更新。假如我们现在建立一个仓库，仓库所在的这个目录中会有一个”.git”的文件夹(ls -al查看)。.git这个文件夹非常重要，所有的版本信息、更新记录，以及Git进行仓库管理的相关信息全都保存在这个文件夹里面。所以，想要删除该目录的git仓库设置，就删除该文件夹即可。

接下来就是使用了，主要就是如下命令按顺序试试，基本就会了:

$git init //在当前目录(工作目录)初始化仓库
#已初始化空的 Git 仓库于 /home/cuixf/abc/.git/
$git status //此时查看工作目录状态，如果有文件会显示
#位于分支 master
#
#初始提交
#
#未跟踪的文件:
#  （使用 "git add <文件>..." 以包含要提交的内容）
#
#	a.txt
#	a/
#
#提交为空，但是存在尚未跟踪的文件（使用 "git add" 建立跟踪）

$git add . //注意有个"."，表示把"./"文件添加到Git里，建立工作空间Workspace，这时如果再次查看，"git status"可以看到新增了文件
$ git commit -m "test1" //这里提交工作目录的当前版本给Git，可以输入此次修改的注释信息如"test1"，可以看到所有文件提交成功:
#[master（根提交） 64f42d0] test1
# 3 files changed, 0 insertions(+), 0 deletions(-)
# create mode 100644 a.txt
# create mode 100644 a/b.txt
# create mode 100644 a/c/c.txt
git status //再次查看，发现目前版本提交完毕
#On branch master
#nothing to commit (working directory clean)

最主要的对文件修改历史进行比较，如果此时已经修改工作目录的文件，修改保存后，输入

$ git status //发现已经出现修改
$ git diff //即可显示修改了什么，会有++,--这样的符号也就是删减情况，另外还可以使用图形GUI界面(gitk命令)看，确认修改后先如上add然后commit即可保存当前版本

另外如果联网后，想要下载/上传自己的仓库，可以看git的命令(clone,push,pull?)，看看github的基本用法。

$git clone/push/pull

继续学习，可以看看多分支branch(git merge/rebase)，看看回滚(git reset/revert/checkout)，看看远程管理(git remote/tag)等等。
